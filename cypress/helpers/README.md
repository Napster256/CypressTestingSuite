# helpers

Директория для вспомогательных модулей и переиспользуемой логики.

Назначение:

- Инкапсуляция повторяющихся операций
- Вынос бизнес-логики из тестовых спецификаций
- Упрощение читаемости тестов
- Централизация утилитарных функций

В helpers размещаются:

- Функции для работы с данными
- Обёртки над API-запросами
- Генераторы тестовых данных
- Общие проверки (assertion helpers)
- Вспомогательные методы для параметризации тестов

Принцип использования:

Тесты должны описывать сценарий, а не реализацию.
Сложная логика выносится в helpers для повышения поддерживаемости и масштабируемости проекта.
